# RouteWolf v51 — DCO включён по умолчанию (с проверкой)

Включает всё из v49 (ночной агент списков) и v50 (фиксы маршрута OpenVPN),
плюс безопасное включение DCO по умолчанию.

## Что изменилось

Раньше конфиг OpenVPN **всегда** получал `disable-dco` — жёстко, без проверки:

```
# Stable default: DCO is disabled because some OpenWrt/X-WRT builds reconnect
# endlessly with ovpn-dco. Use: rw dco on  (or rw dco off)
disable-dco
```

DCO — это обработка пакетов OpenVPN **в ядре**, как у WireGuard/AmneziaWG.
Без него каждый пакет идёт userspace-путём: ядро → tun → процесс openvpn →
шифрование → обратно в ядро → в сокет. Два переключения контекста и копирование
памяти на каждый пакет — отсюда более низкая скорость по сравнению с AmneziaWG.

Сам комментарий в коде объяснял, почему автор перестраховался: на части
сборок OpenWrt/X-WRT ovpn-dco «уходит в бесконечный реконнект». Технически
это несовпадение vermagic — ядро форка собрано не так, как официальный OpenWrt,
и модуль ставится как пакет, но не грузится в ядро.

## Фикс

Тот же принцип, что уже применён к AmneziaWG в этом проекте: **не верить факту
«пакет установлен», проверять, что модуль реально загрузился.**

```sh
dco_kmod_loaded() {
    lsmod 2>/dev/null | grep -qE '^ovpn_dco'
}
```

`install_dco_if_possible()` теперь после установки пакета явно пробует
`modprobe` и подтверждает результат через `lsmod`. Возвращает успех, только
если модуль реально сидит в ядре.

**При установке** (`ovpn_harden_route_only_config`): DCO включается только
если модуль подтверждён загруженным. Не подтверждён — безопасный откат на
`disable-dco`, как раньше. Никакой регрессии для сборок, где DCO не работает.

**`rw dco on`**: раньше писал «DCO enabled/auto» безусловно, даже если модуль
не загрузился, — то есть **врал** об успехе. Теперь при неудаче честно
сообщает об этом и **откатывается на `disable-dco`** вместо того, чтобы
оставить роутер в состоянии, которое и вызывало реконнект-петлю.

**`rw dco status`**: показывает реальное состояние в трёх плоскостях —
что написано в конфиге, загружен ли модуль **прямо сейчас**, и какие пакеты
установлены. Раньше показывал только «есть ли строка disable-dco в файле»,
что не гарантировало ничего про фактическую работу.

## Проверено (сэндбокс-симуляция modprobe/lsmod)

| Сценарий | `install_dco_if_possible` | Конфиг после установки | `dco_set on` |
|---|---|---|---|
| модуль грузится (обычный OpenWrt) | успех | `disable-dco` **отсутствует** | «enabled, verified» |
| модуль не грузится (ImmortalWrt/X-WRT vermagic) | ошибка | `disable-dco` **присутствует** | честно сообщает о неудаче, откат сохранён |

## Как проверить на своём роутере

```sh
rw dco on
rw dco status
```

Должно быть: `Kernel module: loaded (verified) -> DCO is actually active`.
Дальше сравнить скорость загрузки сайтов до/после.

## Что НЕ менялось

`install.sh`, `getdomains-install.sh`, `uninstall.sh`, `cleanup.sh`,
`routewolf-check.sh`, `lists/ipv4.lst`, `lists/ipv6.lst` — не тронуты.
