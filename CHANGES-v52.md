# RouteWolf v52 — версия видна при повторной установке

Включает всё из v49-v51 (списки, роутинг OpenVPN, безопасный DCO) плюс маркер
установленной версии.

## Зачем

Раньше при обнаружении существующей настройки установщик спрашивал
«использовать существующий / заменить», но нигде не показывал, **что именно**
принесёт новый запуск и с какой версии произойдёт переход. Пользователь
выбирал вслепую.

## Что добавлено

`/etc/routewolf/installed_version` — пишется при успешном завершении **любого**
пути установки (`--update`, «использовать существующий», «заменить», чистая
установка). Хранит `PROJECT_VERSION` того установщика, который применялся
последним.

При повторном запуске, если обнаружена существующая настройка, меню теперь
явно показывает:

```
Установленная версия: v51-openvpn-dco-safe -> этот установщик: v52-version-tracking
```

Если маркера нет (роутер настроен до появления этой системы) — сказано прямо,
без пустого/непонятного значения.

Пункт меню 1 переименован из «пропустить настройку туннеля» в **«обновить
код/списки/команды проекта, VPN-конфиг не трогать»** — та же логика, что была
и раньше (`route_vpn()` + `install_management_commands()` вызываются в любом
случае), но теперь явно написано, что произойдёт, а не только в виде
формулировки «пропустить».

## Важно: что этот маркер НЕ делает

Не путать с корневым файлом `version` в репозитории — тот сравнивается
ночным агентом списков (`update.sh --auto`) по содержимому/хешу списков,
это независимый, более старый механизм из v49.

`installed_version` не запускает апгрейд автоматически и не решает за
пользователя. Он только **показывает** дистанцию между тем, что стоит на
роутере, и тем, что несёт установщик. Само решение (обновить компоненты /
заменить конфиг целиком / ничего не делать) по-прежнему принимает человек.

DCO по-прежнему не трогается автоматически при выборе «использовать
существующий» — сознательно: это изменение содержимого рабочего `.ovpn`,
и в этой же сессии мы только что убедились, что DCO может ломать туннель
на конкретном сервере. Включать его — всегда явный шаг: `rw dco on`.

## Проверено

Изолированный тест функций `read_installed_version`/`write_installed_version`:
пустое значение до первой записи, корректное чтение после записи, корректное
чтение СТАРОЙ версии до того, как её перезапишет новая (нужно для меню) —
все три сценария прошли.

Сам экран меню (реальный интерактивный вывод на роутере) не проверялся вживую
в этом раунде — правки только добавляют текст и запись маркера, ветвление
`1/2/3` не менялось.

## Что НЕ менялось

`install.sh`, `getdomains-install.sh`, `uninstall.sh`, `cleanup.sh`,
`routewolf-check.sh`, `lists/ipv4.lst`, `lists/ipv6.lst` — не тронуты.
